木有买域名的情况下，用github+hexo搭建博客遇到的坑

发表于 2017-12-21 | 分类于 Blog | | 阅读次数:

讲述自己木有买域名的情况下，用github+hexo搭建博客遇到的坑

1.由于css或者js路径错误，造成页面在本地localhost上样式没有问题，但是在部署发布后访问发布后的路径没有出现样式的问题

第一步：首先我检查自己的git中项目的设置（Setttings）中Repository name是不是你的命名正确，最好是以你的github的用户名.github.io。


第二步：如果当时在你注册github账号后配置SSH key，那么你在配置_config.yml中的
deploy:
type: git
repository: git@github.com:lvyonglin/lvyonglin.github.io.git
branch: master

repository这个参数的值最好用USE SSH中的地址

第三步：查看自己的git中项目的设置（Setttings）中的GitHub Pages是否配置成功，若是成功则会出现下面的情况

若是没有出现可上述结果则可参照 https://pages.github.com/2,3,4步骤
另外附带转载别人的利用Hexo+github搭建博客，零成本、无需域名、服务器http://blog.csdn.net/wkzd2016/article/details/70170786?locationNum=4&fps=1
在这片文章中关于GitHub Pages的配置可参照 https://pages.github.com/，因为新版的GitHub Pages不支持这篇文章的配置。
